# 🏦 جدير — منصة ذكاء اصطناعي لدعم قرارات تمويل المنشآت

> "سمة تخبرك إذا دفعوا في الماضي — جدير يخبرك إذا يقدرون يدفعون في المستقبل، ولماذا."

---

## ما هو جدير؟

جدير منصة تساعد البنوك على تقييم طلبات تمويل المنشآت الصغيرة والمتوسطة بشكل أسرع وأكثر دقة وشفافية.

بدل ما يجلس موظف الائتمان ساعات يراجع كشوفات يدوياً — جدير يقرأ التدفق النقدي، يحسب درجة ائتمانية ٠–١٠٠، ويشرح **لماذا** بالأرقام. والقرار النهائي يبقى للموظف دائماً.

---

## 🚀 تشغيل سريع

```bash
pip install -r requirements.txt
streamlit run app.py
```

---

## 📁 شرح الملفات

### `app.py` — لوحة موظف الائتمان
الواجهة الكاملة التي يراها موظف البنك. مبنية بـ Streamlit.

تقدر فيها:
- تختار منشأة من القائمة أو تضيف منشأة جديدة
- ترفع كشف حساب CSV
- تشوف الدرجة الائتمانية فوراً مع التفسير الكامل
- تسجّل قرارك: موافقة / مستندات / رفض

---

### `smartlend.py` — قلب المشروع
هنا يحدث كل شيء مهم:

١. يقرأ المعاملات من كشف الحساب
٢. يستخرج **٥ إشارات** مالية:
   - صافي التدفق التشغيلي (مو الدخل الإجمالي — الفرق مهم)
   - استقرار الدخل
   - اتجاه النمو
   - السيولة
   - الإشارات الحمراء (مرتجعات، سحب على المكشوف)
٣. يحسب الدرجة بنموذج XGBoost
٤. يفسّر كل قرار بجمل واضحة للموظف

> 💡 **ميزة مهمة:** نستثني إيداعات المالك من الحساب — المنشأة التي تعيش على ضخّ المالك ليست بصحة جيدة حتى لو الرصيد موجب.

---

### `train_model.py` — تدريب النموذج
يبني نموذج XGBoost على ٤٠٠٠ حالة معايَرة على علاقات مخاطر واقعية.

```bash
python train_model.py
# Accuracy: 76.7% | AUC: 0.805
```

> **ملاحظة صادقة:** البيانات مُصنّعة لأن البيانات السعودية الحقيقية سرّية وتتطلب ترخيص ساما. في الإنتاج، نعيد التدريب على نتائج سداد فعلية.

---

### `credit_model.json` — النموذج المدرّب
ناتج `train_model.py` — يُحمَّل تلقائياً عند تشغيل اللوحة. لا تحذفه 🙂

## 🗺️ الخطة المستقبلية

### المرحلة الأولى — تقوية الأساس

**ربط Open Banking الحقيقي**
نربط مع مزوّد مرخّص من ساما (Lean أو Tarabut) — المنشأة تربط حسابها مباشرة بدون رفع يدوي.

**تكامل سمة**
استعلام تلقائي يظهر جانب درجتنا — الموظف يشوف الصورة كاملة: سجل الماضي (سمة) + قدرة الحاضر (جدير).

**قاعدة بيانات**
نتحوّل من ملفات CSV إلى PostgreSQL — يخلّي المنصة تتعامل مع آلاف الطلبات.

---

### المرحلة الثانية — تعميق الذكاء

**إعادة التدريب على بيانات حقيقية**
بعد ما نجمع معاملات + نتائج سداد فعلية، نعيد التدريب — الدقة ترتفع من ٧٦٪ لـ ٩٠٪+.

**SHAP كامل**
رسم بياني لكل قرار يوضّح: "هذه الإشارة رفعت الخطر، وهذه خفّضته" — بالأرقام.

**تصنيف القطاعات**
مقاولات بدخل موسمي تُقيَّم بمعايير مختلفة عن صيدلية بدخل ثابت.

**كفالة أذكى**
نحسب تلقائياً: هل المنشأة مؤهلة لكفالة؟ كم نسبة الضمان المناسبة؟

---

### المرحلة الثالثة — التوسع والاعتماد

**واجهة المنشأة**
المنشأة ترفع مستنداتها، تتابع طلبها، وتشوف توصيات لتحسين درجتها.

**تكامل مع أنظمة البنك**
API يتكامل مع نظام البنك الأساسي — النتائج تظهر تلقائياً ضمن سير العمل.

**لوحة مدير المخاطر**
توزيع المحفظة، نسب التعثّر المتوقعة، والتنبيهات المبكرة.

**التوجيه الذكي**
الطلبات الكبيرة تروح لموظف خبير، الصغيرة لعام، وطلبات جدة لفرع جدة — تلقائياً.

---

### ما يجعل جدير مختلفاً على المدى البعيد

الأنظمة الموجودة تنظر للوراء — ما حدث. جدير ينظر للأمام — ما سيحدث. هناك منشآت لا تاريخ ائتماني لها، لكن تدفقها النقدي يثبت أنها جديرة بالتمويل. هذه الشريحة لا تخدمها الأنظمة الحالية — وهي سوق ضخم غير مستغَل في السوق السعودي.
