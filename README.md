# 🏦 جدير — منصة ذكاء اصطناعي لدعم قرارات تمويل المنشآت


---

## ما هو جدير؟

جدير منصة تساعد البنوك على تقييم طلبات تمويل المنشآت الصغيرة والمتوسطة بشكل أسرع وأكثر دقة وشفافية.

بدل ما يجلس موظف الائتمان ساعات يراجع كشوفات يدوياً — جدير يقرأ التدفق النقدي، يحسب درجة ائتمانية ٠–١٠٠، ويشرح **لماذا** بالأرقام. والقرار النهائي يبقى للموظف دائماً.

---

## 🚀 تشغيل سريع

```bash
pip install -r requirements.txt
streamlit run app.py
```
link: https://nuhaai-jadeer.hf.space/
---


أربعة كشوف حساب جاهزة للتجربة: test_clothing.csv و test_restaurant.csv و test_consulting.csv و test_shipping.csv
ارفع أي منها في اللوحة عبر "➕ إضافة شركة جديدة" وشوف النموذج يقيّمها فوراً.


## 📁 شرح الملفات

### `app.py` — لوحة موظف الائتمان
الواجهة الكاملة التي يراها موظف البنك. مبنية بـ Streamlit.

تقدر فيها:
- تختار منشأة من القائمة أو تضيف منشأة جديدة
- ترفع كشف حساب CSV
- تشوف الدرجة الائتمانية فوراً مع التفسير الكامل
- تسجّل قرارك: موافقة / مستندات / رفض.

---

### `smartlend.py` — قلب المشروع
هنا يصير كل شيء مهم:

١. يقرأ المعاملات من كشف الحساب

٢. يستخرج **٥ إشارات** مالية:
   - صافي التدفق التشغيلي (مو الدخل الإجمالي — الفرق مهم)
   - استقرار الدخل
   - اتجاه النمو
   - السيولة
   - الإشارات الحمراء

٣. يحسب الدرجة بنموذج XGBoost

٤. يفسّر كل قرار بجمل واضحة للموظف


---

### `train_model.py` — تدريب النموذج
يبني نموذج XGBoost على ٤٠٠٠ حالة معايَرة على علاقات مخاطر واقعية.

```bash
python train_model.py
# Accuracy: 76.7% | AUC: 0.805
```

---

### `credit_model.json` — النموذج المدرّب
ناتج `train_model.py` — يُحمَّل تلقائياً عند تشغيل اللوحة. لا تحذفه 🙂

# الرؤية المستقبلية

## المرحلة الأولى — تقوية الأساس

* **ربط Open Banking الحقيقي**
  التكامل مع مزودي المصرفية المفتوحة المرخصين من ساما مثل Lean أو Tarabut لجلب البيانات مباشرة دون رفع يدوي.

* **تكامل مع سمة**
  عرض التقرير الائتماني إلى جانب تقييم جدير للحصول على رؤية تجمع بين السجل الائتماني السابق والقدرة المالية الحالية.

* **قاعدة بيانات احترافية**
  الانتقال من ملفات CSV إلى PostgreSQL لدعم عدد أكبر من الطلبات وتحسين الأداء.

---

## المرحلة الثانية — تطوير الذكاء

* **إعادة تدريب النموذج على بيانات حقيقية**
  استخدام بيانات سداد فعلية عند توفرها لتحسين دقة النموذج مع مرور الوقت.

* **دمج SHAP**
  عرض تأثير كل مؤشر مالي على القرار الائتماني بطريقة مرئية وقابلة للتفسير.

* **نماذج خاصة بكل قطاع**
  إنشاء نماذج مختلفة للقطاعات مثل المقاولات، التجزئة، والصحة لأن طبيعة التدفقات النقدية تختلف بينها.

* **توصيات ذكية لبرنامج كفالة**
  اقتراح أهلية المنشأة لبرنامج كفالة ونسبة الضمان المناسبة تلقائيًا.

---

## المرحلة الثالثة — التوسع

* **بوابة إلكترونية للمنشآت**
  تمكّن المنشآت من رفع البيانات، متابعة الطلب، والحصول على توصيات لتحسين الجدارة الائتمانية.

* **تكامل مع أنظمة البنوك**
  توفير API يربط جدير مباشرة مع أنظمة البنوك لتسريع اتخاذ القرار.

* **لوحة لإدارة المخاطر**
  متابعة المحافظ الائتمانية، توقع نسب التعثر، والتنبيه المبكر للمخاطر.

* **توزيع ذكي للطلبات**
  توجيه الطلبات تلقائيًا إلى الموظف أو الفرع المناسب حسب نوع الطلب ومستوى المخاطر.

---


> **لا يهدف جدير إلى استبدال الأنظمة الحالية، بل إلى تعزيزها بذكاء اصطناعي قابل للتفسير يساعد جهات التمويل على اتخاذ قرارات أسرع وأكثر اتساقًا، ويفتح فرص التمويل أمام المنشآت الواعدة التي تمتلك تدفقات نقدية قوية حتى وإن كان تاريخها الائتماني محدودًا.**
